## Why does an L1 penalty on sigmoid gates encourage sparsity?

The **sigmoid function** maps each learnable gate score to a value in (0, 1).
Adding the **L1 norm** (sum of all gate values) to the loss directly penalises
the network for every gate that stays open — even slightly.

The gradient of the L1 term with respect to a gate value *g* is a **constant ±1**,
regardless of how close *g* already is to zero.
This means the optimiser always receives the same push toward zero and can drive
gates to *exactly* 0 — unlike L2 regularisation, whose gradient shrinks near zero
and only produces small-but-nonzero values.

This property (promoting **exact sparsity**) is the same reason LASSO regression
is preferred over Ridge when you want truly sparse solutions.

The **λ hyperparameter** controls the trade-off:

| λ setting | Effect |
|-----------|--------|
| Low (e.g. 1e-5) | Weak pruning pressure; network stays dense, highest accuracy |
| Medium (e.g. 1e-4) | Balanced; meaningful sparsity with modest accuracy drop |
| High (e.g. 1e-3) | Aggressive pruning; highest sparsity, potential accuracy loss |

## Gate Value Distribution Plot

After training, `gate_distribution.png` shows a histogram of **all gate values**
for the best-performing model.

A successful self-pruning outcome will exhibit two clear features:

1. **A large spike near 0** — the majority of weights have been pruned (their gates
   driven to near zero by the L1 penalty).
2. **A smaller cluster away from 0** (typically 0.5 – 1.0) — the surviving weights
   that the network identified as important for classification.

## Analysis

- **Low λ (1e-5):** Minimal pruning pressure. The network retains most weights,
  achieving the highest accuracy but lowest sparsity — behaves close to an
  ordinary unpruned network.

- **Medium λ (1e-4):** The "sweet spot". Meaningful sparsity is achieved with
  only a modest accuracy drop, demonstrating that many weights were redundant.

- **High λ (1e-3):** Aggressive pruning. Sparsity is highest, but accuracy may
  degrade as the network is forced to discard potentially useful weights.

This confirms the fundamental **sparsity–accuracy trade-off** inherent to
magnitude/gate-based pruning methods.

## Implementation Notes

| Component | Choice | Reason |
|-----------|--------|--------|
| Gate activation | Sigmoid | Smooth, differentiable, output ∈ (0,1) |
| Sparsity loss | L1 norm of gates | Promotes exact zeros (unlike L2) |
| Optimizer | Adam | Adaptive LR; handles gate_scores and weights jointly |
| LR schedule | Cosine Annealing | Smooth decay, avoids sharp LR cliffs |
| Prune threshold | 1e-2 | Gates < 0.01 considered effectively zero |
